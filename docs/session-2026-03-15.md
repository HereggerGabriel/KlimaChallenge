# TravelApp — Session Log 2026-03-15

## Session 3 — User Page Overhaul + Trip Detail + Fixes

### 1. TripDetailModal (`components/ui/TripDetailModal.tsx`)

New modal that opens when the user taps any trip card in the Recent Trips list.

- Displays route title, +XP badge, date, time, transport type, cost, distance, and note.
- Header has two icon buttons (top-right): `delete-outline` and `close`.
- Trash icon shows an in-app styled confirm overlay (dark Palette theme) warning about XP deduction before deleting.
- On confirm: trip is removed, XP is subtracted (min 0), modal closes.
- **Icon note:** Uses `MaterialIcons` from `@expo/vector-icons/MaterialIcons` — NOT `IconSymbol` (SF Symbols). SF Symbols do not render reliably inside React Native `<Modal>` on iOS.

### 2. XP Seeding Fix

- `loadTrips()` in `services/tripStorage.ts` now coerces `distance` and `cost` to float via `parseFloat(...) || 0` on parse, preventing NaN propagation from old or corrupt AsyncStorage data.
- On first app load (or if stored XP is missing/`"NaN"`), `user.tsx` seeds XP by summing `getXPForTrip()` across all loaded trips, then persists the result.

### 3. Logout Button

- Replaced hidden icon-only logout with a clearly labeled, red-bordered button at the bottom of the user screen.
- On press: clears auth and routes to `/onboarding`.

### 4. User Page Overhaul (`app/(tabs)/user.tsx`)

- `paddingTop: 52` on the header for status bar safe area.
- Centered `ActivityIndicator` during the loading state.
- Section title icons: star for Favorites, clock for Recent Trips.
- Trip cards have a left-side colored accent bar per transport type (Bus=blue.mid, Train=green.mid, Tram=blue.light, Subway=green.dark).
- Recent Trips defaults to 5 entries; "Show all X / Show less" toggle.

## Session 8 — Quests, Achievements & Main Quest

### 8. Quest System (`utils/questSystem.ts`) _(new)_

Core data model and helpers for the quests feature.

- `QUESTS_PER_ROTATION = 3` — quests shown per period (of a larger pool).
- `MAIN_QUEST_ID`, `MAIN_QUEST_XP = 2000`, `MAIN_QUEST_CELEBRATED_KEY` — main quest constants.
- `pickRandomQuests(pool, n)` — random shuffle + slice; used for daily/weekly rotation.
- `getClaimKey(quest)` — daily → `id:YYYY-MM-DD`, weekly → `id:monday-date`, milestone → `id`.
- `timeUntilMidnight()` / `daysUntilMonday()` — used in section subtitles.
- **Daily pool** (8 quests): log trip, train, bus, tram, 10km, 15km, 2 trips, multi-modal.
- **Weekly pool** (8 quests): 5/7 trips, 3/4 transport types, 50/100km, 3 trains, 5-day spread.
- **Milestone quests** (9): first trip, 10/25/50/100 trips, multi-modal pioneer, 100/500/1000km.

### 9. Achievement System (`utils/achievementSystem.ts`) _(new)_

- `computeStreak(trips)` — consecutive days with ≥1 trip, ending today or yesterday.
- `computeSavedVsCar(trips)` — `Σ(distance × 0.30 − cost)`.
- **19 achievements** across 5 categories:
  - General (4): first trip, 5/25/100 trips
  - Distance (4): 10/100/500/1000km total
  - Transport (5): train×5, bus×5, tram×5, subway×3, all-4-types
  - Streaks (3): 3/7/30 consecutive days
  - Savings (3): €20/€100/€500 saved vs car

### 10. Quests & Achievements Screen (`app/quests.tsx`) _(new)_

Single screen with segmented control (Quests | Achievements tabs).

**Quest rotation logic:**

- `loadOrRefreshSelection(storageKey, periodKey, pool)` — checks stored `{period, ids}` in AsyncStorage; regenerates with `pickRandomQuests` if period has changed (new day/new week start). Stable within a period, random across periods.

**Quests tab:**

- `MainQuestFeaturedCard` at top — full-width card (red border/shadow, `Palette.blue.mid` bg), showing trophy icon, title, €X / €Y stat, red progress bar, % covered label, and CLAIM 2000 XP button when complete.
- Daily section (3 of 8, resets at midnight) with section count.
- Weekly section (3 of 8, resets Monday) with section count.
- Milestones section (all 9, one-time) with section count.
- `QuestCard` — icon, title, XP badge (color-coded by type), description, progress bar, CLAIM button or check.

**Achievements tab:**

- Global total row: `claimedAchievements.size / ACHIEVEMENTS.length`.
- Per-category section headers with claimed/total counts.
- `AchievementCard` — 2-column grid, category-colored icon with lock overlay for incomplete, progress bar, XP badge, CLAIM/claimed state.

**XP integration:**

- `handleClaimQuest`, `handleClaimAchievement`, `handleClaimMainQuest` — each persists to AsyncStorage and calls `awardXP(amount)`.
- `awardXP` — updates XP state, fires `XPToast`, fires `LevelUpOverlay` on level-up.

**Navigation:** `useLocalSearchParams<{ tab? }>()` → sets initial tab; enables deep-link from profile page to achievements tab.
